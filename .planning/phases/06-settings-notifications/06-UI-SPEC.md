## Color

The app uses a warm dark-gray monochrome surface. Color exclusively signals money (income/expense). All values come directly from `globals.css`.

| Role            | Token                                        | Raw Value                | Usage                                                                      |
| --------------- | -------------------------------------------- | ------------------------ | -------------------------------------------------------------------------- |
| Dominant (60%)  | `--color-paper` / `bg-background`            | `oklch(0.268 0.004 100)` | Page background, settings page surface                                     |
| Secondary (30%) | `--color-card` / `bg-card`                   | `oklch(0.3 0.005 100)`   | Settings card container, notification dropdown panel background            |
| Raised surface  | `--color-raised` / `bg-accent`               | `oklch(0.315 0.005 100)` | Unread notification row highlight, active nav state                        |
| Accent (10%)    | `--color-ink` / `text-primary`               | `oklch(0.952 0.005 90)`  | Primary save button fill, active bell icon state                           |
| Destructive     | `--color-expense` / `text-destructive`       | `oklch(0.68 0.18 25)`    | No destructive actions in this phase (no deletes)                          |
| Hairline        | `--color-hairline` / `border`                | `oklch(0.36 0.005 100)`  | Card borders, section separators, dropdown panel border                    |
| Ink-soft        | `--color-ink-soft` / `text-muted-foreground` | `oklch(0.76 0.008 90)`   | Help text, notification message body, timestamps, read notification titles |

**Accent reserved for:**

- Primary "Save Settings" button fill (only CTA on settings page)
- Bell icon when it has unread notifications (swap to `text-foreground` from `text-muted-foreground`)
- Unread notification row background: `bg-accent` (same as active nav state — `--color-raised`)

**Unread badge:** `bg-destructive text-destructive-foreground` — uses the semantic destructive token for the numeric count badge (matches shadcn `Badge` `variant="destructive"`). This is a deliberate choice: destructive in this context means "attention required", not "danger", consistent with industry-standard notification badge convention.

**Money color rule (preserved from Phases 2–4):** No income/expense colors appear on the settings page or notification rows — only on monetary amounts if any notification message displays a currency value (it does: "₱X from Salary expected today"). Currency amounts in notification messages are `text-income` (`oklch(0.72 0.13 155)`) — same rule as income amount display in Phase 2.

**No pure extremes:** No `#000` or `#fff` anywhere. All values from the monochrome ramp.

**Source:** `apps/web/src/app/globals.css` @theme block

## Component Inventory

### Settings Page — `/settings`

**Page layout:** Single settings page at `(dashboard)/settings/page.tsx`. Two card sections stacked vertically: "Display Currency" above "Income Reminders". Max width `max-w-2xl` (matches income page `max-w-4xl` narrowed for a form-focused layout — settings forms are narrow by nature).

**Page heading:** `<h1 className="text-[20px] font-semibold leading-tight mb-6">Settings</h1>`

**Section card pattern:** Same `rounded-xl border bg-card shadow-sm p-6 flex flex-col gap-4` as `pf-percentage-editor.tsx` — no deviation.

#### Section 1: Display Currency

```
[Card]
  Section heading: "Display Currency"  (text-[20px] font-semibold)
  [Separator]
  Help text: "Choose how monetary amounts are displayed across the app. This is display-only — no conversion is applied."
             (text-sm text-muted-foreground)
  [Label] Currency
  [Select — currency selector]
    Options (in order):
    - PHP — Philippine Peso (₱) [default, selected on first load]
    - USD — US Dollar ($)
    - EUR — Euro (€)
    - GBP — British Pound (£)
    - SGD — Singapore Dollar (S$)
    - AUD — Australian Dollar (A$)
    - JPY — Japanese Yen (¥)
    - CAD — Canadian Dollar (C$)
  Select trigger width: full width of card body
```

#### Section 2: Income Reminders

```
[Card]
  Section heading: "Income Reminders"  (text-[20px] font-semibold)
  [Separator]
  Help text: "Receive a reminder email on your chosen schedule. Reminders are sent in Manila time (UTC+8)."
             (text-sm text-muted-foreground)

  Row: [Switch] [Label "Enable reminders"]
       Switch uses existing shadcn Switch component (already in income-form.tsx)
       Default state: off (unchecked) for new users

  --- Conditional section shown only when switch is ON ---

  [Label] Frequency
  [Select — frequency]
    Options: Daily | Weekly | Monthly
    Default: Daily

  Conditional sub-fields (shown based on frequency):
    WEEKLY only:  [Label "Day of week"]  [Select: Sunday|Monday|Tuesday|Wednesday|Thursday|Friday|Saturday]
    MONTHLY only: [Label "Day of month"] [Select: 1st through 28th] (capped at 28 to avoid short-month issues)
                  Help text below: "Reminders are sent on this day each month."

  [Label] Hour (Manila time)
  [Select — hour]
    Options: 12:00 AM (midnight) through 11:00 PM in 1-hour steps (24 items)
    Format: "9:00 AM", "12:00 PM", "6:00 PM" — 12-hour format with AM/PM suffix
    Default: "9:00 AM"

  --- End conditional section ---

  [Button variant="default"] Save Settings   (self-start, same pattern as expense-form)
```

**Conditional reveal animation:** None — sections appear/disappear with `hidden`/`block` class swap via React state. No animation — matches the no-bounce motion rule (only `rise` and `fill` keyframes exist in globals.css).

**Save button state:** Disabled with `opacity-50` while server action is in flight. Loading label: "Saving…" — matches existing pattern in pf-percentage-editor and expense-form.

### Notification Center

**Pattern decision:** Bell icon + `DropdownMenu` panel in `DashboardNav`. Not a dedicated page. Justification: reference service caps list at 50; at that volume a dropdown is conventional. A dedicated `/notifications` page is unnecessary for v1.

#### Bell Icon in DashboardNav

Position in nav: rightmost element in the nav bar, after the existing 5 route links. Added as a separate `div` with `ml-auto` to push it to the right side of the `max-w-7xl` container.

```
<DropdownMenu>
  <DropdownMenuTrigger asChild>
    <Button variant="ghost" size="icon" className="relative h-8 w-8 shrink-0" aria-label="Notifications">
      <Bell className="h-4 w-4" />                    <!-- lucide Bell icon -->
      {unreadCount > 0 && (
        <Badge
          variant="destructive"
          className="absolute -top-1 -right-1 h-4 w-4 p-0 flex items-center justify-center text-[10px] font-medium tabular-nums"
        >
          {unreadCount > 9 ? '9+' : unreadCount}
        </Badge>
      )}
    </Button>
  </DropdownMenuTrigger>
  <DropdownMenuContent align="end" className="w-80">
    <!-- NotificationList panel -->
  </DropdownMenuContent>
</DropdownMenu>
```

**Unread count cap:** Display `9+` when count exceeds 9 — matches platform conventions (iOS, Gmail).

**Bell icon active state:** When `unreadCount > 0`, bell button uses `text-foreground` instead of `text-muted-foreground`. When `unreadCount === 0`, `text-muted-foreground` — same visual treatment as inactive nav links.

#### NotificationList Panel

Rendered inside `DropdownMenuContent` (`w-80`, `align="end"`):

```
Panel structure:
  Header row: "Notifications" (text-sm font-semibold)  |  "Mark all read" (Button variant="ghost" size="sm" text-xs)
              shown only when at least 1 unread item exists; hidden when all read
  [Separator]
  Scrollable list (max-h-[400px] overflow-y-auto):
    For each notification (newest first, max 50):
      [NotificationItem row]
    If no notifications:
      [Empty state]
  [Separator]
  Footer: "Settings" link → /settings  (text-xs text-muted-foreground, text-center)
```

**NotificationItem row:**

```
Row className:
  - Unread: "flex flex-col gap-0.5 px-3 py-3 cursor-pointer rounded-md bg-accent/50 hover:bg-accent"
  - Read:   "flex flex-col gap-0.5 px-3 py-3 cursor-pointer rounded-md hover:bg-accent/50 text-muted-foreground"

Contents:
  Title row: [dot indicator] [title text]
    Unread dot: "h-1.5 w-1.5 rounded-full bg-primary shrink-0 mt-1.5" (small filled circle)
    Read dot: hidden (no dot)
    Title: text-sm font-medium (unread) | text-sm text-muted-foreground (read)
  Message:  text-xs text-muted-foreground, 2-line truncate (line-clamp-2)
  Timestamp: text-xs text-muted-foreground tabular-nums (e.g. "Jun 6, 9:00 AM")
```

**Click behavior (D-08):** Clicking any notification row calls `markAsRead(id)` then navigates to `notification.link` via `router.push()`. The dropdown closes. If `notification.link` is null, clicking only marks as read without navigation.

**Mark-all-read:** Button in panel header. Only visible when `unreadCount > 0`. Calls `markAllAsRead()` then optimistically updates local state to mark all items as read. Shows `"Marking…"` during pending state.

**Unread count refresh:** Fetched server-side on each page load (SSR in `DashboardNav` parent layout). After `markAsRead` / `markAllAsRead` calls, trigger `router.refresh()` to rehydrate unread count without a full page reload.

## Interaction Contracts

### Settings Form — Save Flow

1. User changes a setting field
2. User clicks "Save Settings"
3. Button shows "Saving…" and is disabled
4. On success: `toast.success("Settings saved.")` — matches Phase 2–4 success toast pattern
5. On error: `toast.error("Could not save settings. Please try again.")` — matches Phase 2–4 error toast
6. After save: `router.refresh()` to rehydrate server-fetched settings

### Reminder Toggle — Conditional Reveal

1. Switch OFF (default): frequency, day, hour pickers are hidden (not mounted — avoids submitting empty values)
2. Switch toggled ON: frequency/day/hour section appears immediately (no animation delay)
3. Switch toggled OFF again: section disappears; form submits `reminderEnabled: false`; server clears all schedule fields

### Notification Click — Mark Read + Navigate

1. User clicks notification row
2. Optimistic update: row switches to read style immediately (no waiting for API round-trip)
3. API call: `PUT /api/notifications/:id/read` fires in background
4. Router: `router.push(notification.link)` — navigates without waiting for API
5. If API call fails: `toast.error("Could not mark notification as read.")` — does not block navigation

### Notification Mark-All-Read

1. User clicks "Mark all read" in panel header
2. All rows in list switch to read style immediately (optimistic)
3. API call: `PUT /api/notifications/read-all`
4. Panel header "Mark all read" button disappears (all read state)
5. Bell badge disappears
6. On API error: `toast.error("Could not mark all notifications as read.")` and `router.refresh()` to revert optimistic state

## States Required

### Settings Page

| State         | Display                                                                        |
| ------------- | ------------------------------------------------------------------------------ |
| Loading (SSR) | Page skeleton — use Next.js loading.tsx with same card shape, muted background |

### Notification Bell

| State                | Display                                              |
| -------------------- | ---------------------------------------------------- |
| No notifications     | Bell icon `text-muted-foreground`, no badge          |
| Unread count 1–9     | Bell icon `text-foreground`, badge shows exact count |
| Unread count 10+     | Bell icon `text-foreground`, badge shows "9+"        |
| Dropdown open        | `DropdownMenuContent` visible, full list rendered    |
| Loading mark-as-read | Row optimistically styled as read; spinner not shown |

### NotificationList

| State                         | Display                                                                                                                |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| Empty (no notifications ever) | Empty state in panel body                                                                                              |
| All read                      | No "Mark all read" button; all rows in read style; bell badge absent                                                   |
| Mixed                         | Unread rows at top with `bg-accent/50`; read rows below in muted style (sorted by createdAt DESC — not by read status) |
| Marking all read              | Button shows "Marking…" disabled; rows optimistically go muted                                                         |

## Accessibility Contract

- Bell icon `<Button>` must have `aria-label="Notifications"` (screen readers read icon-only buttons)
- `<DropdownMenuContent>` receives focus on open; arrow keys navigate items; Escape closes
- Settings `<Switch>` must have associated `<Label>` with `htmlFor` pointing to switch id — same pattern as income-form.tsx
- All `<Select>` components must have `<Label>` with `htmlFor` → SelectTrigger `id`
- Unread notification dot is `aria-hidden="true"` — read state is conveyed by the title text weight and color alone (not exclusively by the dot)
- Touch targets: all interactive elements minimum 32px height — bell button is `h-8 w-8` (32px), notification rows are `py-3` (12px × 2 + content ~20px = ~44px)

---

## Motion Contract

- No new animations introduced
- Conditional section reveal (reminders fields): immediate show/hide — no transition (no animation for toggled content in any existing Phase 2–4 component)
- Notification dropdown: Radix `DropdownMenu` uses its built-in Tailwind fade — `data-[state=open]:animate-in data-[state=closed]:animate-out` (shadcn default, already in the component)
- No bounce, no elastic — matches the `ease-out exponential only` rule in globals.css
