<objective>
Cron/email producer slice (NOTIF-02): an hourly Cloudflare Workers cron trigger whose `scheduled` handler (a) finds users whose reminder schedule matches the current Manila hour/day and sends them a reminder email via Resend while mirroring it as an INCOME_REMINDER in-app notification (D-05), and (b) creates one-time PENDING_INCOME_DUE in-app notifications for pending incomes due today in Manila (D-06/D-07, never emailed). This closes the loop: Plan 02 stores the schedule, Plan 03 displays the notifications, this plan produces them.

Purpose: Delivers the second success criterion (scheduled reminder emails) and the production source for the notification center. Idempotency, Manila-time correctness, and honoring the user's display currency in notification copy (SET-01) are the load-bearing concerns.

Output: extended EmailService with `sendIncomeReminderEmail`, a dependency-free Manila-time helper, the cron-service `runCron`, the Module Worker `export default { fetch, scheduled }` change, and the hourly cron trigger in wrangler.toml.
</objective>
