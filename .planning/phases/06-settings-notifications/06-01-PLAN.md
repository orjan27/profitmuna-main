<objective>
Foundation slice for Phase 6: extend the Drizzle schema with the `notifications` table, user settings columns, and an `incomes` dedup column; generate and apply the additive D1 migration; keep the test DB helper DDL in sync; create failing test scaffolds for settings/notifications/cron; and upgrade `formatCurrency` to be currency-aware without changing existing call sites' output.

Purpose: All three downstream slices (settings, notification center, cron) depend on these schema objects, contract types, and the migrated database. Without the applied migration, build/type checks pass on schema-derived types while the live DB lacks the tables — a false-positive verification state. This plan removes that risk before any feature work.

Output: Migrated local D1 database, synced test DDL, shared web contract types, currency-aware formatter, and RED test scaffolds.
</objective>
