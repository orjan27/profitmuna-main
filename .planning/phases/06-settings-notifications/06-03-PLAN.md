## Trust Boundaries

| Boundary                        | Description                                                                 |
| ------------------------------- | --------------------------------------------------------------------------- |
| browser → BFF proxy → Hono      | Notification reads + mark-read mutations cross network; auth cookie relayed |
| notification id param → service | Untrusted :id could target another user's notification                      |

## STRIDE Threat Register

| Threat ID | Category               | Component                  | Disposition | Mitigation Plan                                                                                                                                                            |
| --------- | ---------------------- | -------------------------- | ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-6-06    | Elevation of privilege | markAsRead by :id          | mitigate    | service WHERE includes BOTH eq(id) AND eq(userId) — a user cannot mark or read another user's notification (IDOR, ASVS V4); acceptance criteria asserts the dual predicate |
| T-6-07    | Elevation of privilege | all notification endpoints | mitigate    | `.use('/*', requireAuth)` at router level — no list/count/mark without valid JWT                                                                                           |
| T-6-08    | Tampering              | GET / query params         | mitigate    | zValidator('query', notificationQuerySchema) bounds limit 1..50 and coerces unreadOnly before service                                                                      |
| T-6-09    | Information disclosure | notification list          | mitigate    | list query scoped `eq(notifications.userId, userId)` — never returns another user's notifications                                                                          |
| T-6-SC    | Tampering              | npm installs               | accept      | No new dependencies (all shadcn components already installed per UI-SPEC §Registry Safety)                                                                                 |

</threat_model>
