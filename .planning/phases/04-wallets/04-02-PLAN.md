<objective>
The wallet-creation and balance vertical slice: a user can create a PROFIT_FIRST or BLANK wallet, map income/expense categories with conflict-blocking, choose the 3-mode expense behavior, and see all wallets as a card grid with the derived total balance — including an empty state that one-click-creates wallets for unlinked allocation accounts.

This delivers WAL-01 (create), WAL-02 (mapping + auto-deduct toggle), and WAL-03 (list with computed balances). After this plan a real user can create wallets and watch balances compute from PF allocation + mapped income − mapped expenses. Manual transactions and the detail view come in Plan 03.

Purpose: Land the authoritative wallet service (CRUD + mapping conflict enforcement + balance computation ported from the reference) and the create/list UI on top of the Plan 01 spine.
Output: createWalletService with list/create/update/remove + mapping helpers + balance math; four route handlers; the wallet list page (card grid + empty state + quick-create); the create form (type/color/category/3-mode expense); server actions; WAL-01/02/03 unit tests.
</objective>
