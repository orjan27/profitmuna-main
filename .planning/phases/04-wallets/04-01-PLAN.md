## Trust Boundaries

| Boundary                     | Description                                                                      |
| ---------------------------- | -------------------------------------------------------------------------------- |
| browser → Next.js BFF proxy  | user input (query params, JSON bodies) crosses into the same-origin proxy        |
| Next.js server → Workers API | access token forwarded as Bearer; API is the real authorization point            |
| Workers API → D1             | all persisted wallet mutations; userId scoping enforced in service (Plans 02/03) |
| deploy → wallet migration    | one-time additive DDL over the live D1                                           |

## STRIDE Threat Register

| Threat ID | Category               | Component                                      | Disposition | Mitigation Plan                                                                                                                                                   |
| --------- | ---------------------- | ---------------------------------------------- | ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-04-01   | Spoofing               | wallet routes without auth                     | mitigate    | `walletsRouter.use('/*', requireAuth)` at router level (Task 2); proxy forwards Bearer from httpOnly access_token cookie                                          |
| T-04-02   | Information Disclosure | CORS reflecting wildcard origin                | mitigate    | CORS keeps existing `origin: c.env?.APP_BASE_URL ?? ''`; only verifies allowMethods includes PUT/DELETE/PATCH — no wildcard introduced (Task 2)                   |
