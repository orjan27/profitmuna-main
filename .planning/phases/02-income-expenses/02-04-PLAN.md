<objective>
Category management vertical slice for both income and expenses (INC-06, EXP-05): lazy idempotent seeding of system defaults (D-04), create/rename/delete of custom categories surfaced where they are used (D-11), cascade rename of the denormalized categoryName (D-13), block-delete-in-use (D-12), and protection of system defaults — DB → API → UI.

Output: income + expense category services (factory, seeding, cascade, protection), category routes, unit tests, and a "manage categories" dialog (plus quick-add affordance) reachable from each list page and record form.
</objective>
