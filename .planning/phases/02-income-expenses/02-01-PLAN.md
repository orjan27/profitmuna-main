<threat_model>

## Trust Boundaries

| Boundary                             | Description                                                           |
| ------------------------------------ | --------------------------------------------------------------------- |
| browser → Next.js server actions/RSC | user input (form data, query params) crosses into server-side fetches |
| Next.js server → Workers API         | access token forwarded; API is the real authorization point           |
| Workers API → D1                     | all persisted mutations; userId scoping enforced here                 |

## STRIDE Threat Register

| Threat ID | Category               | Component                                  | Disposition | Mitigation Plan                                                                                                         |
| --------- | ---------------------- | ------------------------------------------ | ----------- | ----------------------------------------------------------------------------------------------------------------------- |
| T-02-01   | Spoofing               | mounted routes without auth                | mitigate    | Every route group mounted behind `app.use('/api/<resource>/*', requireAuth)` in index.ts (Task 2)                       |
| T-02-02   | Information Disclosure | CORS reflecting wildcard origin            | mitigate    | CORS keeps existing `origin: c.env?.APP_BASE_URL ?? ''`; only `allowMethods` expanded (Task 2) — no wildcard introduced |
| T-02-03   | Tampering              | seeding race creating duplicate categories | mitigate    | Unique index on (userId, name) on both category tables (Task 1) enables onConflictDoNothing in Plan 04                  |
| T-02-04   | Tampering              | float/precision errors on money            | mitigate    | amount stored as integer cents; `toCents` helper centralizes conversion (Task 3)                                        |
| T-02-SC   | Tampering              | npm/pip/cargo installs                     | accept      | No new packages installed this phase (RESEARCH §Package Legitimacy Audit: all deps pre-pinned); no install task         |

</threat_model>
