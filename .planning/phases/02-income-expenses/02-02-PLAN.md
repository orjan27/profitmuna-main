## Trust Boundaries

| Boundary                | Description                                             |
| ----------------------- | ------------------------------------------------------- |
| browser → server action | form data (amount, categoryId, dates) crosses to server |
| Next.js → Workers API   | access token forwarded; API authorizes                  |

## STRIDE Threat Register

| Threat ID | Category               | Component                                     | Disposition | Mitigation Plan                                                                                                        |
| --------- | ---------------------- | --------------------------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------- |
| T-02-05   | Elevation of Privilege | IDOR — reading/mutating another user's income | mitigate    | every income-service query includes `eq(incomes.userId, userId)`; scoped miss → 404 not 403 (Task 1, asserted by test) |
| T-02-06   | Tampering              | negative or fractional amount                 | mitigate    | Zod `amount.positive()`; stored as integer cents via toCents (Task 1/2)                                                |
| T-02-07   | Tampering              | categoryId belonging to another user          | mitigate    | resolveCategoryName validates category ownership before insert/update; else 400 invalid_category (Task 1)              |
| T-02-08   | Tampering              | mass-assign profitFirstAllocated on receive   | mitigate    | receive() never writes profitFirstAllocated; receiveIncomeSchema only accepts receivedDate (Task 1)                    |
| T-02-09   | Spoofing               | unauthenticated income access                 | mitigate    | /api/incomes/\* mounted behind requireAuth (Plan 01); /income page getSession→redirect (Task 2)                        |
| T-02-SC   | Tampering              | npm installs                                  | accept      | No new packages this phase                                                                                             |

</threat_model>
