## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                            |
| ----- | ----- | ---------------------------------------------------------------- |
| xs    | 4px   | Icon gaps, color swatch gaps, inline badge padding               |
| sm    | 8px   | Compact field labels, progress bar height, card action spacing   |
| md    | 16px  | Default element spacing, form field gap, card padding supplement |
| lg    | 24px  | Card internal padding (shadcn Card default), section gap         |
| xl    | 32px  | Layout column gap, page section vertical gap                     |
| 2xl   | 48px  | Page top padding, major section breaks                           |
| 3xl   | 64px  | Full-page layout spacing                                         |

Exceptions:

- Color swatch palette grid: 6px gap between swatches (design need; acceptable between xs and sm).
- Progress bar height: 8px (sm token).
- Amount masking toggle button: min touch target 44px height (accessibility — icon-only button, matches shadcn `size="icon"` at `size-9` = 36px + ensure 44px touch area via padding on mobile).

**Source:** shadcn Card default padding (24px), existing LoginForm gap-4 pattern (16px), standard 8-point grid.

## Typography

| Role    | Size | Weight         | Line Height | Usage                                                     |
| ------- | ---- | -------------- | ----------- | --------------------------------------------------------- |
| Body    | 14px | 400 (regular)  | 1.5         | Account descriptions, filter labels, card metadata        |
| Label   | 14px | 500 (medium)   | 1.4         | Form field labels, card subtitles, category chip text     |
| Heading | 20px | 600 (semibold) | 1.2         | Page heading ("Profit First"), card titles (account name) |
| Display | 28px | 600 (semibold) | 1.1         | Derived balance amount per account card                   |

Notes:

- Percentage values in the bulk editor inputs: 16px / weight 400 / line-height 1.5 (Body-class but rendered in `<Input>`).
- Amount in masked state (`••••`): same Display size; replace text content only, not font metrics.
- All monetary amounts (`₱X,XXX.XX`) use the Display role; masked amounts use the same slot.
- Shadcn `CardTitle` renders `font-semibold leading-none` — aligns with Heading at 20px.

**Source:** Existing shadcn new-york defaults + `card.tsx` (`font-semibold`) + auth form patterns (14px labels, 14px descriptions via `text-sm`).

## Color

| Role            | Value                     | Usage                                                                             |
| --------------- | ------------------------- | --------------------------------------------------------------------------------- |
| Dominant (60%)  | `hsl(var(--background))`  | Page background, page-level surface                                               |
| Secondary (30%) | `hsl(var(--card))`        | Account cards, percentage editor panel, filter sheet, dialog surface              |
| Accent (10%)    | `hsl(var(--primary))`     | Primary CTA button, active filter badge, focused input ring, "Add Account" button |

Accent reserved for:

- "Add Account" primary action button
- "Save Percentages" primary action button
- Active date-range preset button (selected state)
- Active/selected income category filter chips
- Focused input ring (Tailwind focus-visible)

Account-card color accents (left border or color swatch indicator) use the per-account `color` field directly (hex from `PF_DEFAULT_COLORS`):

- `#10b981` — Profit (emerald)
- `#8b5cf6` — Owner Pay (violet)
- `#f59e0b` — Tax (amber)
- `#f43f5e` — Operating Expenses (rose)
- `#3b82f6` — blue (custom)
- `#ec4899` — pink (custom)
- `#14b8a6` — teal (custom)
- `#f97316` — orange (custom)

These per-account hex colors are data-driven, not design system tokens. They appear ONLY as:

- 4px left border accent on account cards
- 12px filled circle swatch in the color picker
- Color dot indicator next to account name in the percentage editor row

**Source:** `apps/web/src/components/ui/button.tsx` (variant tokens), `card.tsx` (`bg-card`), CONTEXT.md D-03/D-08, RESEARCH.md `PF_DEFAULT_COLORS`.

## Layout Patterns

### Page Layout: `/profit-first`

```
┌─────────────────────────────────────────────────────┐
│  Page heading ("Profit First")          [+ Add Account] [Eye toggle] │
│  ─────────────────────────────────────────────────  │
│  Filter bar: [Date range ▼]  [Categories ▼]  [Edit %] │
│  ─────────────────────────────────────────────────  │
│  Account cards grid (2-col on md+, 1-col on mobile) │
│  ┌─────────────┐  ┌─────────────┐                   │
│  │ [●] Profit  │  │ [●] Owner   │                   │
│  │ 5%          │  │ Pay 50%     │                   │
│  │ ₱X,XXX.XX   │  │ ₱X,XXX.XX   │                   │
│  │ [=========] │  │ [=========] │  [⋮]              │
│  └─────────────┘  └─────────────┘                   │
│  … (Tax, OPEX, custom accounts)                     │
└─────────────────────────────────────────────────────┘
```

- Page is an RSC (`page.tsx`); reads `searchParams` for `from`, `to`, `categoryIds`.
- Filter bar and account cards are client components that use `useQueryState` from nuqs.
- "Edit %" opens the inline bulk percentage editor panel (replaces the cards area or slides in from side — match reference `pf-percentage-editor.tsx` behaviour).

### Account Card

```
┌─────────────────────────────────────────┐
│ [color-bar] Account Name     [CUSTOM] [⋮]│
│             Target: 5%                  │
│             ₱X,XXX.XX  (or ••••••)       │
│             [=======================]   │
│             of ₱X,XXX.XX total income   │
└─────────────────────────────────────────┘
```

- Color bar: 4px left border in account's `color` hex.
- Badge: shown for non-CUSTOM accounts with type label (PROFIT / OWNERS_PAY / TAX / OPEX); hidden for CUSTOM.
- Dropdown [⋮]: Edit / Delete (Delete disabled for non-CUSTOM; disabled state shown with `disabled` attribute + `opacity-50`).
- Progress bar: `value = targetPercentage / 100` (0–100 range).

### Bulk Percentage Editor Panel

```
┌─────────────────────────────────────────┐
│  Edit Percentages               [Cancel] │
│  ─────────────────────────────────────  │
│  [●] Profit           [  5  ]  %        │
│  [●] Owner Pay        [ 50  ]  %        │
│  [●] Tax              [ 15  ]  %        │
│  [●] Operating Exp.   [ 30  ]  %        │
│  [●] Custom Account   [  0  ]  %        │
│  ─────────────────────────────────────  │
│  Total: 100% ✓  (or 97% ✗ — live count) │
│                          [Save Changes]  │
└─────────────────────────────────────────┘
```

- Total line: green text + checkmark when exactly 100%; destructive-colored text + × when != 100%.
- Save button: disabled when total != 100%.
- Inputs: number input, integer only (0–100), `min=0 max=100`.

### Create/Edit Account Dialog

```
┌──────────────────────────────┐
│  Add Account         [✕]     │
│  ─────────────────────────── │
│  Account Name                │
│  [_____________________]     │
│                              │
│  Target %                    │
│  [___]  %                    │
│                              │
│  Color                       │
│  [●][●][●][●][●][●][●][●]    │
│  (8 preset swatches)         │
│                              │
│           [Cancel] [Save]    │
└──────────────────────────────┘
```

### Delete Confirmation Dialog

```
┌──────────────────────────────┐
│  Delete Account      [✕]     │
│  ─────────────────────────── │
│  Delete "Account Name"?      │
│  This cannot be undone.      │
│                              │
│     [Cancel]  [Delete]       │
└──────────────────────────────┘
```

- "Delete" button uses `variant="destructive"`.
- Dialog is NOT shown for non-CUSTOM accounts (dropdown entry is visually disabled).

## Interaction States

| Element              | State                 | Visual Treatment                                                                 |
| -------------------- | --------------------- | -------------------------------------------------------------------------------- |
| Account card         | Default               | `bg-card`, `shadow-sm`, `rounded-xl`                                             |
| Account card         | Hover                 | `shadow-md` transition (subtle elevation change)                                 |
| Percentage input     | Valid                 | default border                                                                   |
| Percentage input     | Error / exceeds 100%  | `aria-invalid` → destructive ring (via shadcn Input)                             |
| Save button          | Total != 100%         | `disabled` → `opacity-50 pointer-events-none`                                    |
| Delete dropdown item | Non-CUSTOM account    | `disabled` with tooltip "Default accounts cannot be deleted"                     |
| Delete dropdown item | Wallet-linked account | `disabled` with tooltip "This account is linked to a wallet" (Phase 4 activates) |
| Amount toggle        | Visible               | Eye-off icon (lucide `EyeOff`); tooltip "Hide amounts"                           |
| Amount toggle        | Hidden                | Eye icon (lucide `Eye`); tooltip "Show amounts"                                  |
| Amount               | Masked                | Renders `••••••` in same typographic slot                                        |
| Filter date preset   | Selected              | `variant="default"` button (filled accent)                                       |
| Filter date preset   | Unselected            | `variant="outline"` button                                                       |
| Category filter chip | Selected              | `variant="default"`                                                              |
| Category filter chip | Unselected            | `variant="outline"`                                                              |
| Total line (editor)  | == 100%               | `text-green-600` + `✓`                                                           |
| Total line (editor)  | != 100%               | `text-destructive` + `✗`                                                         |

## Amount Masking (Shared Component)

Component: `apps/web/src/components/amount-visibility.tsx`

Exports:

- `useAmountVisibility()` — hook: `{ visible: boolean, toggle: () => void, mounted: boolean }`
- `AmountToggle` — icon-button that calls `toggle()`; uses `Eye` / `EyeOff` lucide icons
- `MaskedAmount` — renders `₱X,XXX.XX` when `mounted && visible`, renders `••••••` otherwise

SSR safety: default `visible = false`; guard with `mounted` flag (see RESEARCH.md Pattern code + Pitfall 7). SSR always renders masked state to avoid hydration mismatch.

Storage key: `'pf-amounts-visible'` in `localStorage`.
