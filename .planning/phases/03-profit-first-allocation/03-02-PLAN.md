## STRIDE Threat Register

| Threat ID | Category                      | Component                               | Disposition | Mitigation Plan                                                                                                                                                                                                                 |
| --------- | ----------------------------- | --------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-03-04   | Elevation of Privilege (IDOR) | GET summary, PATCH/DELETE accounts/:id  | mitigate    | Every Drizzle WHERE includes `eq(profitFirstAccounts.userId, c.get('userId'))` and income queries filter `userId`; deleteAccount/updateAccount return 404 when the row's userId ≠ caller (Task 1 behavior + Security Domain V4) |
| T-03-05   | Tampering                     | targetPercentage / basis-point overflow | mitigate    | Zod `z.number().int().min(0).max(10000)` at route boundary on create/update/percentages (V5)                                                                                                                                    |
| T-03-06   | Tampering                     | account name / color injection          | mitigate    | name `z.string().max(100).trim()`; color `z.enum(PF_DEFAULT_COLORS)` rejects arbitrary hex; React renders names as text content (no dangerouslySetInnerHTML)                                                                    |
| T-03-07   | Elevation of Privilege        | delete-guard bypass via direct API call | mitigate    | `accountType !== 'CUSTOM'` enforced in the service layer (not UI-only); BFF/route both pass through requireAuth                                                                                                                 |
| T-03-08   | Information Disclosure        | stale derived balance after % change    | mitigate    | Balances always derived server-side per request; never persisted; web plans call router.refresh() on mutation                                                                                                                   |
| T-03-09   | Spoofing                      | unauthenticated access to PF endpoints  | mitigate    | `profitFirstRouter.use('/*', requireAuth)` guards every route; BFF forwards Bearer token only                                                                                                                                   |
| T-03-SC   | Tampering                     | npm installs                            | accept      | No new packages (RESEARCH Package Legitimacy Audit: none); no install tasks                                                                                                                                                     |
