## Trust Boundaries

| Boundary                                 | Description                                                                  |
| ---------------------------------------- | ---------------------------------------------------------------------------- |
| registration request → auth-service      | Untrusted email/password/name; seeding runs server-side after user insert    |
| Google OAuth callback → upsertGoogleUser | Provider-verified identity; seeding must run only on genuine new-user branch |
| deploy → backfill migration              | One-time DB mutation over existing user rows                                 |

## STRIDE Threat Register

| Threat ID | Category                           | Component                            | Disposition | Mitigation Plan                                                                                                                                                                                      |
| --------- | ---------------------------------- | ------------------------------------ | ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| T-03-01   | Tampering                          | seedProfitFirstAccounts userId scope | mitigate    | Seeding always uses the freshly-inserted `user.id`/`inserted[0].id` from the same transaction context — never client-supplied; accounts scoped to that userId only                                   |
| T-03-02   | Information Disclosure / Integrity | Google branch seeding                | mitigate    | Seed ONLY in branch 3 (brand-new user); branches 1/2 are returning users — seeding them would duplicate rows and violate unique(userId,name). Enforced by placement + unit test "does not duplicate" |
| T-03-03   | Integrity                          | Backfill migration re-run            | mitigate    | `NOT IN (SELECT DISTINCT user_id FROM profit_first_accounts)` guard makes the backfill idempotent (Pitfall 6) — no duplicate seeding on redeploy                                                     |
| T-03-SC   | Tampering                          | npm/pip/cargo installs               | accept      | No new packages introduced this phase (RESEARCH Package Legitimacy Audit: none). No install tasks.                                                                                                   |
