## Trust Boundaries

| Boundary                     | Description                                                 |
| ---------------------------- | ----------------------------------------------------------- |
| browser → /profit-first page | URL search params (from/to/categoryIds) are user-controlled |
| RSC SSR fetch → API          | Forwards the session access_token as a Bearer header        |
| rendered account names → DOM | Stored strings rendered into the page                       |

## STRIDE Threat Register

| Threat ID | Category               | Component                            | Disposition | Mitigation Plan                                                                                                                       |
| --------- | ---------------------- | ------------------------------------ | ----------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| T-03-06   | Tampering (Stored XSS) | account name rendering in cards      | mitigate    | React renders `account.name` as text content only; no dangerouslySetInnerHTML anywhere in pf-overview                                 |
| T-03-08   | Information Disclosure | stale masked/derived balances        | mitigate    | Balances come from the per-request RSC fetch; nuqs URL change / router.refresh() re-fetches; nothing cached client-side beyond render |
| T-03-09   | Spoofing               | unauthenticated page access          | mitigate    | middleware.ts redirects to /login when no tokens; SSR fetch only sends a valid Bearer when present                                    |
| T-03-10   | Tampering              | malformed from/to/categoryIds in URL | mitigate    | Server-side summaryQuerySchema (Plan 02) validates query params; nuqs parsers coerce types client-side                                |
| T-03-SC   | Tampering              | shadcn component installs            | mitigate    | All nine components from the shadcn OFFICIAL registry (UI-SPEC Registry Safety: no third-party); no npm dependency additions          |
