<decisions_resolved>

- **Generic responses** on forgot-password (D-discretion enumeration mitigation): always 200, identical
  body, whether or not the email exists. Only send the email if a user actually exists.
- **Reset token reuse of the auth_tokens machinery** built in slice 01 (purpose 'reset_password',
  expiresAt now+1h per D-10). Single-use: DELETE on redemption.
- **Reset rate limiting:** reuse the login_attempts table keyed by email for forgot-password throttling
  (security.md: rate-limit email-sending endpoints), or a lightweight per-email cooldown; document choice.
- **Post-reset session invalidation:** setting a new password deletes all refresh_tokens for the user
  (forces re-login everywhere — limits damage if the account was compromised).
  </decisions_resolved>

## STRIDE Threat Register

| Threat ID | Category               | Component                                 | Disposition | Mitigation Plan                                                                           |
| --------- | ---------------------- | ----------------------------------------- | ----------- | ----------------------------------------------------------------------------------------- |
| T-03-01   | Spoofing               | reset link token replay                   | mitigate    | sha256-hashed single-use token, DELETE on redemption, 1h expiry (D-09/D-10)               |
| T-03-02   | Information disclosure | email enumeration on forgot-password      | mitigate    | Always-generic 200 response; email sent only when user exists; identical UI confirmation  |
| T-03-03   | Tampering              | reset-link request flooding (email abuse) | mitigate    | Per-email cooldown via login_attempts (security.md rate-limit on email-sending endpoints) |
| T-03-04   | Elevation of privilege | stale sessions after password change      | mitigate    | resetPassword deletes all refresh_tokens for the user (force re-login everywhere)         |
