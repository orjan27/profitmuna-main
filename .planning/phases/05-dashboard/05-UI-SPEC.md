## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                                                            |
| ----- | ----- | ------------------------------------------------------------------------------------------------ |
| xs    | 4px   | Icon gaps, inline badge padding, feed row icon-to-text gap, stat card icon alignment             |
| sm    | 8px   | Compact element spacing, nav link padding, stat card sub-label gap, date preset button gap       |
| md    | 16px  | Default element spacing, form field gap, card padding supplement, stat card content internal gap |
| lg    | 24px  | Card internal padding (shadcn Card default), section gap, page header-to-content gap             |
| xl    | 32px  | Dashboard section vertical gap (stat section to PF section to feed section)                      |
| 2xl   | 48px  | Page top padding, major section breaks                                                           |
| 3xl   | 64px  | Full-page layout spacing, sidebar nav height allocation                                          |

Exceptions:

- Feed row touch target: natural flow with `py-3` (12px) gives approximately 44px touch target at body font size — satisfies mobile accessibility minimum without a fixed height rule.
- Nav sidebar link items: minimum 44px height via `py-3 px-4` padding on mobile.
- Stat card icon container: 48px × 48px (`h-12 w-12`) icon wrapper — intentionally 48px (12×4) for visual weight; not a grid spacing token.
- Date filter preset button gap: 6px (`gap-1.5`) — inherited directly from `pf-filters.tsx` `gap-1.5` pattern; treated as an approved exception.

**Source:** Phase 4 UI-SPEC spacing contract (inherited unchanged). Exceptions derived from existing `pf-filters.tsx` + `amount-visibility.tsx` + `income-list.tsx` patterns.

## Typography

| Role    | Size | Weight         | Line Height | Usage                                                                                              |
| ------- | ---- | -------------- | ----------- | -------------------------------------------------------------------------------------------------- |
| Body    | 14px | 400 (regular)  | 1.5         | Feed row descriptions, category names, nav sub-labels, stat card sub-labels, date strings          |
| Label   | 14px | 400 (regular)  | 1.4         | Section sub-headings, stat card labels — differentiated from Body via `text-muted-foreground`      |
| Heading | 20px | 600 (semibold) | 1.2         | Page heading ("Dashboard"), section headings ("Profit First", "Recent Transactions"), nav headings |
| Display | 28px | 600 (semibold) | 1.1         | Stat card primary values (₱0.00 total income, etc.), Total Wallet Balance card value               |

Notes:

- Stat card amount values use the Display role (28px semibold). Net Income conditional color styling (destructive red when negative) applies to the Display role without changing font properties.
- Feed row amounts: 14px / weight 600 (semibold) / tabular-nums — slightly heavier than Body for financial scannability, but same size. This is a usage-level override, not a new type role.
- Nav item labels: Body role (14px / 400) for inactive links; Heading role (14px / 600) for the active link — same size, weight change only.
- `MaskedAmount` renders `••••••` in the same typographic slot as the formatted amount — no size change on masking.
- Getting-started hint CTAs on empty feed: Body role (14px / 400); CTA link text uses accent color, not a weight change.

**Source:** Phase 4 UI-SPEC typography contract (inherited). Confirmed against `pf-overview.tsx` (`text-[20px] font-semibold` for CardTitle, `text-[28px] font-semibold` for MaskedAmount display) and `income-list.tsx` (`text-sm font-medium` = 14px/500 — note: executor should use 400/600 split per Phase 4 spec; existing 500 uses in Phase 2/3 are grandfathered).

## Color

| Role            | Value                     | Usage                                                                                                   |
| --------------- | ------------------------- | ------------------------------------------------------------------------------------------------------- |
| Dominant (60%)  | `hsl(var(--background))`  | Page background, page-level surface, nav sidebar background                                             |
| Secondary (30%) | `hsl(var(--card))`        | Stat cards, PF account cards (dashboard read-only), feed section container, nav hover/active background |
| Accent (10%)    | `hsl(var(--primary))`     | Active date preset button, active nav link, getting-started CTA links, feed "Load more" button          |
| Destructive     | `hsl(var(--destructive))` | Negative Net Income stat card amount (text only); no destructive action buttons on the dashboard        |

Accent reserved for:

- Active/selected date preset filter button (`variant="default"` — filled accent)
- Active nav link in the sidebar (active state indicator)
- Getting-started CTA link buttons in the empty feed state (e.g. "Record your first income")
- "Load more" button in the recent transactions feed (`variant="outline"` with accent focus ring, or `variant="default"` — see Interaction States)

Destructive color reserved for:

- Net Income stat card amount text when `netIncomeCents < 0` — text color only, no button styling
- No confirm/delete dialogs exist on the dashboard page itself; destructive dialogs are on Phase 2/4 pages

```

│  [Logo / Brand]                                             │
│  ─────────────────────────────────────────────────────────  │
│  ┌──────────────────┐  ┌──────────────────────────────────┐ │
│  │ Sidebar Nav      │  │ <main> content (route children)  │ │
│  │ ──────────────── │  │                                  │ │
```

- Sidebar: fixed left, `w-64` on desktop; collapses to hamburger/icon-only on mobile (`md:hidden` overlay or drawer).

- Layout outer container: `min-h-screen bg-background flex`. Sidebar: `w-64 shrink-0 border-r bg-card`. Main: `flex-1 pt-8 pb-16 px-6 md:px-8 max-w-5xl`.

### PF Accounts Section (Dashboard Read-Only)

- Section heading: "Profit First" with a `→` link to `/profit-first` (text button `variant="ghost"` or plain `<Link>`).
- Cards: same `grid grid-cols-1 md:grid-cols-2 gap-6` as `/profit-first` page.

### Feed Row

```
[type badge]  label (category/wallet name)  description  ₱amount  date
```

- Container: `<Link href={tx.href}>` wrapping a flex row: `flex items-center gap-3 py-3 px-4 hover:bg-muted/40 rounded-lg transition-colors`.
- Type badge: `<Badge variant="secondary" className="shrink-0 w-20 justify-center text-xs capitalize">`. Values: "Income", "Expense", "Deposit", "Withdrawal".
- Label: `<span className="flex-1 text-sm truncate">` — category name for income/expense, wallet name for wallet_deposit/wallet_withdrawal.
- Description (optional): `<span className="text-xs text-muted-foreground truncate max-w-[8rem]">` — hidden if null.
- Amount: `<MaskedAmount>` with `className="tabular-nums text-sm font-semibold {amountClass}"` where `amountClass` is `text-emerald-600` for income/deposit, `text-rose-600` for expense/withdrawal.
- Date: `<span className="text-xs text-muted-foreground w-20 text-right">` — `formatDate(tx.date)`.
- Rows are `<Link>` elements navigating to: income → `/income`, expense → `/expenses`, wallet_deposit/wallet_withdrawal → `/wallets/{walletId}`.
- No edit/delete actions on feed rows. Navigation-only (D-05).
- Feed list wrapper: `<div className="divide-y rounded-lg border">` — matches `income-list.tsx` container.

### "Load More" Button

- Positioned below the feed list, centered: `<div className="flex justify-center mt-4">`.
- Button: `variant="outline"` with label "Load more".
- When loading next page: `disabled` + spinner icon, text "Loading…".
- When `hasMore` is false: button is hidden (not disabled — remove from DOM).
- Pagination is client-side state (`useState` in `DashboardContent`) — NOT a URL param, to avoid full RSC re-render (per RESEARCH.md Pitfall 5 / A1).
