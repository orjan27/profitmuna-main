## Trust Boundaries

| Boundary           | Description                                                                                   |
| ------------------ | --------------------------------------------------------------------------------------------- |
| browser → Next RSC | searchParams (from/to) are untrusted; passed only to the API which Zod-validates them         |
| client → BFF proxy | Load more fetch crosses /api/dashboard/\* same-origin; BFF forwards the httpOnly access_token |

## STRIDE Threat Register

| Threat ID | Category               | Component              | Disposition | Mitigation Plan                                                                                                                                 |
| --------- | ---------------------- | ---------------------- | ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| T-05-06   | Spoofing               | /dashboard route       | mitigate    | middleware.ts already redirects unauthenticated users to /login; page reads httpOnly access_token cookie, never trusts client-supplied identity |
| T-05-07   | Tampering              | from/to in fetch URL   | mitigate    | Forwarded to the API which validates them with the Zod date regex (T-05-02); UI never builds SQL                                                |
| T-05-08   | Information Disclosure | Load more client fetch | mitigate    | Goes through the BFF proxy which attaches the user's own Bearer token; API scopes results to that userId (T-05-03)                              |
| T-05-09   | Information Disclosure | Amount masking         | accept      | MaskedAmount renders ●●●●●● on SSR (mounted guard) — values never leak before the user toggles visibility; cosmetic, not a security control     |

</threat_model>
